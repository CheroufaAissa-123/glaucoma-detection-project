# Branch Workflow

This project uses `main` as the stable branch.

Each contributor should work on a dedicated `feature-*` branch and keep changes focused on one responsibility at a time.

## Recommended Steps

1. Start from the latest `main`
2. Create or switch to your personal feature branch
3. Commit small logical changes
4. Push your branch to GitHub
5. Open a pull request only when the work is ready for review
